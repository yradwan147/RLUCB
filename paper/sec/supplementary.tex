\section{Experimental Details}
\label{sec:supp_exp}

\paragraph{Hardware and Runtime.}
All experiments were conducted on a standard consumer laptop (Apple M-series CPU, 16 GB RAM).
A single experiment run (100 students per group, 10,000 questions per student) completes in under 60 seconds.
The three replicate runs used seeds 42, 10042, and 20042.
No GPU was required.

\paragraph{Software Dependencies.}
The simulation is implemented in Python 3.11 using the following packages: NumPy ($\geq$1.24.0) for numerical computation, Gymnasium ($\geq$0.29.0) for the RL environment interface, Matplotlib ($\geq$3.7.0) and Seaborn ($\geq$0.12.0) for visualization, Pandas ($\geq$2.0.0) for data export, and tqdm ($\geq$4.65.0) for progress reporting.

\paragraph{Student Initialization.}
Category difficulties $d_i$ are sampled from $\text{Uniform}(0.3, 0.7)$ using the experiment seed, ensuring identical difficulties across both groups.
Each student's initial knowledge is sampled independently using a per-student seed (base seed + student index), and both groups share the same per-student seeds to ensure identical initial conditions.
The control group's random selector uses an offset seed (student seed + 1,000,000) to decorrelate selection randomness from student randomness.

\paragraph{UCB Initialization.}
The UCB selector performs round-robin initialization: any category with zero attempts is selected before the UCB score formula is applied.
With $C = 6$ categories, this means the first 6 questions for each student cycle through all categories before adaptive selection begins.

\section{Limitations}
\label{sec:supp_lim}

\begin{itemize}
    \item \textbf{Simulated students only.} All results are based on simulated students with simplified learning and forgetting dynamics. Real human learning involves complex cognitive processes (motivation, fatigue, prerequisite dependencies) not captured by our model. Validation with actual student data is essential before deployment.
    \item \textbf{Stationary category difficulties.} We assume fixed category difficulties throughout the experiment. In reality, learning one topic may facilitate or hinder learning in related topics (transfer effects).
    \item \textbf{Single forgetting model.} We use a simple exponential decay model. More sophisticated forgetting models (e.g., ACT-R memory model, half-life regression) may produce different dynamics.
    \item \textbf{Limited strategy comparison.} We compare only UCB1 against uniform random selection. Other bandit algorithms (Thompson Sampling, $\varepsilon$-greedy, EXP3) and contextual bandits that incorporate the student's current knowledge state may perform differently.
    \item \textbf{Fixed hyperparameters.} The exploration parameter $c = \sqrt{2}$ is the standard UCB1 value. Tuning $c$ for educational settings (e.g., higher $c$ for uncertain learners) is an open question.
    \item \textbf{Category-level granularity.} We select at the category level with 6 categories. Finer-grained selection (individual questions or skills) and larger numbers of categories would test scalability.
\end{itemize}
