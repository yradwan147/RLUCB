# UCB vs Random Quizzing Experiment

A reinforcement learning experiment comparing **UCB-based adaptive quizzing** against **random quizzing** for simulated student learning.

## Overview

This experiment simulates two groups of students:
- **UCB Group**: Questions selected using the Upper Confidence Bound algorithm (adaptive, focuses on weak areas)
- **Control Group**: Questions selected uniformly at random

The simulation tracks how each group's "knowledge" (modeled as per-category success probabilities) evolves over time, with realistic learning and forgetting dynamics.

```

```

### Population Settings

| Argument | Short | Default | Description |
|----------|-------|---------|-------------|
| `--students` | `-s` | 50 | Number of students per group |
| `--categories` | `-c` | 6 | Number of knowledge categories |

### Learning Dynamics

| Argument | Short | Default | Description |
|----------|-------|---------|-------------|
| `--learning-rate` | `-lr` | 0.10 | How much knowledge increases on correct answer (alpha) |
| `--penalty` | | 0.03 | How much knowledge decreases on incorrect answer (beta) |

**Learning formula:**
- Correct answer: `k_new = k + alpha * (1 - k)` (asymptotic toward 1.0)
- Incorrect answer: `k_new = k - beta * k` (small penalty)

### Forgetting Dynamics

| Argument | Default | Description |
|----------|---------|-------------|
| `--decay-rate` | 0.02 | Exponential decay rate per timestep |
| `--base-knowledge` | 0.15 | Minimum retained knowledge after forgetting |

**Forgetting formula (Ebbinghaus-inspired):**
```
k_new = base + (k - base) * exp(-decay_rate * time_since_exposure)
```

### Experiment Settings

| Argument | Short | Default | Description |
|----------|-------|---------|-------------|
| `--questions` | `-q` | 200 | Number of questions per student session |
| `--exploration` | | 1.414 | UCB exploration parameter (c), typically sqrt(2) |
| `--seed` | | 42 | Random seed for reproducibility |
| `--runs` | | 1 | Number of experiment runs (for statistical analysis) |

### Output Settings

| Argument | Short | Default | Description |
|----------|-------|---------|-------------|
| `--output` | `-o` | `results` | Output directory for saved files |
| `--dashboard` | | False | Generate comprehensive dashboard (all plots in one figure) |
| `--individual-plots` | | False | Generate separate plot files |
| `--csv` | | False | Export results to CSV file |
| `--no-show` | | False | Don't display plots (only save to files) |
| `--quiet` | `-Q` | False | Suppress progress bars and summary output |

## Understanding the Results

### Key Metrics

- **Average Knowledge**: Mean probability of correct answer across all categories
- **Knowledge Variance**: How uniform knowledge is across categories (lower = more balanced)
- **Weakest Category**: Knowledge level in the student's worst category
- **Exposure Distribution**: How many times each category was quizzed

### Expected UCB Advantages

1. **Faster weak area improvement**: UCB focuses on categories where students struggle
2. **More uniform knowledge**: Lower variance across categories
3. **Adaptive behavior**: Shifts focus as weaknesses are addressed

### When UCB Shines

- Longer sessions (200+ questions)
- Higher learning rates
- When initial knowledge varies significantly across categories

## Programmatic Usage

```python
from experiment.config import ExperimentConfig
from experiment.simulation import Experiment
from experiment.visualization import create_dashboard, print_summary_statistics

# Configure experiment
config = ExperimentConfig(
    num_students_per_group=50,
    num_categories=6,
    questions_per_session=200,
    learning_rate=0.10,
    decay_rate=0.02,
    random_seed=42,
)

# Run experiment
experiment = Experiment(config)
results = experiment.run()

# Print summary
print(print_summary_statistics(results))

# Generate dashboard
create_dashboard(results, save_path="dashboard.png")

# Export to CSV
results.save_to_csv("results.csv")
```
